# Brief Compiler - Testing & Running Guide

## Prerequisites

Install Rust (if not already installed):

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env
```

Verify installation:
```bash
rustc --version
cargo --version
```

## Building

```bash
# Build the project
cargo build

# Build in release mode (optimized)
cargo build --release

# Run tests
cargo test
```

## Running the Compiler

### Basic Usage

```bash
# Run on a .bv file (default: type check + proof verify + execute)
cargo run -- sample.bv

# Verbose output (shows each stage)
cargo run -- sample.bv --verbose
cargo run -- sample.bv -v

# Skip type checking (faster for development)
cargo run -- sample.bv --skip-typecheck

# Skip proof verification (faster for development)
cargo run -- sample.bv --skip-proof
```

### Available Options

| Option | Description |
|--------|-------------|
| `-a`, `--annotate` | Generate path annotations as comments |
| `--verbose` / `-v` | Show detailed output for each compiler stage |
| `--skip-typecheck` | Skip type checking phase |
| `--skip-proof` | Skip proof verification phase |

### Path Annotation

Use `-a` or `--annotate` to generate annotated code showing all call paths:

```bash
cargo run -- sample.bv -a
```

Annotations are wrapped in a collapsible block marked by `=== PATH ANALYSIS ===` and `=== END PATH ANALYSIS ===`. This allows you to:
- Filter them out: `grep -v '^// ==='`
- Fold them in editors that support comment folding

Example output:
```brief
# My Brief program
# Entry point below

// === PATH ANALYSIS ===
// Annotations generated by brief-compiler --annotate
// PATHS_TO: main
// CALLERS: (none)
// [ENTRY_POINT]
rct txn main [true][done == true] {
  init();
  setup();
  term;
};

// PATHS_TO: main → init
// CALLERS: main
defn init(): Void [true][true] {
  term;
};
// === END PATH ANALYSIS ===
```

**Annotation types:**
- **PATHS_TO**: All paths that lead to each function
- **CALLERS**: Who calls each function
- **[ENTRY_POINT]**: Entry points (reactors with `[true]` precondition)
- **[RECURSIVE]**: Functions that call themselves

## Compiler Pipeline

The compiler runs through these stages in order:

1. **Lexer** (`src/lexer.rs`) - Tokenizes source code
2. **Parser** (`src/parser.rs`) - Builds AST from tokens
3. **Type Checker** (`src/typechecker.rs`) - Verifies type safety
4. **Proof Engine** (`src/proof_engine.rs`) - Verifies contracts and proofs
5. **Interpreter** (`src/interpreter.rs`) - Executes the program

## Writing Test Programs

Create a `.bv` file with Brief code:

```brief
// Example: sample_test.bv
let counter: Int = 0;

rct txn increment [counter < 10][counter == @counter + 1] {
  &counter = counter + 1;
  term;
};
```

Run it:
```bash
cargo run -- sample_test.bv --verbose
```

## Expected Output

### Successful Run
```
[Lexer] Tokenizing...
[Parser] Successfully parsed 3 items
[TypeChecker] Running type checks...
[TypeChecker] No type errors
[ProofEngine] Running proof verification...
[ProofEngine] All proofs verified
[Interpreter] Running program...
[Interpreter] Final state: {...}
Execution completed successfully
```

### Error Cases

**Parse Error:**
```
Parse error: Expected ;, found identifier
```

**Type Error:**
```
Type errors found:
  - TypeMismatch { expected: "Int", found: "String", context: "let count" }
```

**Proof Error:**
```
Proof errors found:
  - NoAcceptingPath { txn: "my_txn" }
```

## Debugging

### Enable Debug Logging
```bash
RUST_LOG=debug cargo run -- sample.bv
```

### Check AST Output
The parser prints the full AST when running. To inspect just the AST:

```bash
cargo run -- sample.bv 2>&1 | grep -A 100 "Parsed program"
```

### Common Issues

1. **Missing semicolons**: Brief requires `;` after every statement
2. **Contract syntax**: Must have both `[pre]` and `[post]` conditions
3. **Unification**: Pattern matching requires `Type(value) = expr;` syntax

## Testing New Features

When adding new compiler features:

1. Add lexer tokens in `src/lexer.rs`
2. Add AST nodes in `src/ast.rs`
3. Add parser rules in `src/parser.rs`
4. Add type checking in `src/typechecker.rs`
5. Add proof verification in `src/proof_engine.rs`
6. Add interpretation in `src/interpreter.rs`
7. Add test file(s) in the project root
8. Run `cargo test` to verify

```
│   ├── ast.rs          # AST node definitions

│   ├── main.rs         # CLI entry point

│   ├── proof_engine.rs # Contract & exhaustiveness verification

│   └── typechecker.rs  # Type inference & verification
├── sample.bv           # Example Brief program
├── Cargo.toml          # Rust project config
└── TESTING.md          # This file
```
